  ## About

  This project is a C++ simulation of four fundamental CPU scheduling algorithms used in modern Operating Systems. Rather than just implementing one algorithm, the simulator runs all four on the same process set and produces a comparative analysis — making it a practical study tool for understanding OS scheduling trade-offs.

  Built as part of an Operating Systems course project, this deepened understanding of how real-world schedulers balance throughput, fairness, and deadline guarantees.

<Section id="challenge">
  ## Challenge

  Implement all four major CPU scheduling paradigms in a single unified simulator:

  1. **FCFS (First Come First Serve)** — the simplest non-preemptive baseline.
  2. **Round Robin (RR)** — preemptive with a configurable time quantum.
  3. **Rate Monotonic Scheduling (RMS)** — priority-based for real-time periodic tasks.
  4. **Earliest Deadline First (EDF)** — dynamic priority for hard real-time systems.

  Then visualize the scheduling sequence for each and compare metrics like average waiting time, turnaround time, and CPU utilization.
</Section>

<Section id="solution">

## Solution

<AppSolution href={appData.url}>

**Unified Process Model:** Defined a `Process` struct holding burst time, arrival time, period, and deadline — flexible enough to work across all four scheduling models without duplication.

<ul role="list" className="mt-6 space-y-8">
  <li>
    <span>
      <strong className="font-semibold text-blue-100">FCFS:</strong> Non-preemptive queue implementation. Processes execute in arrival order to completion. Simple but shows how convoy effect degrades performance with mixed burst-time processes.
    </span>
  </li>
  <li>
    <span>
      <strong className="font-semibold text-blue-100">Round Robin:</strong> Circular ready queue with a configurable time quantum (Q). Processes are preempted after Q units and placed back at the tail — balancing fairness with context-switch overhead, visible in the comparison output.
    </span>
  </li>
  <li>
    <span>
      <strong className="font-semibold text-blue-100">Rate Monotonic Scheduling (RMS):</strong> Static priority assignment — shorter period = higher priority. A preemptive priority queue drives execution. The simulator validates schedulability using the Liu & Layland bound (U ≤ n(2^(1/n) − 1)).
    </span>
  </li>
  <li>
    <span>
      <strong className="font-semibold text-blue-100">Earliest Deadline First (EDF):</strong> Dynamic priority — the process with the nearest absolute deadline always runs next. Priority queue is re-evaluated at every preemption point. Achieves 100% CPU utilization for feasible task sets.
    </span>
  </li>
  <li>
    <span>
      <strong className="font-semibold text-blue-100">Sequence Visualization & Comparison:</strong> Each algorithm outputs a Gantt-style text timeline showing execution order. A final comparison table prints average waiting time, turnaround time, and CPU utilization side-by-side for the same process set.
    </span>
  </li>
</ul>
