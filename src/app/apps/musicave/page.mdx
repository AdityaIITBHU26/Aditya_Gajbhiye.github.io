<Section id="challenge">
  ## Challenge

  The goal was to build a complete, production-like music streaming experience — not just a basic CRUD app. Key requirements included:

  1. Stream and download songs and albums without page reloads.
  2. Show popular albums and allow users to search for songs, albums, and artists.
  3. Recommend songs in real-time based on what is currently playing.
  4. Include digital wellbeing features — listening time awareness and session limits.
  5. Support a shuffle mode that plays random songs independently of playlist history.
</Section>

<Section id="solution">

## Solution

<AppSolution href={appData.url}>

**Backend Architecture:** Built with Node.js and Express.js, exposing a clean REST API for all music data — songs, albums, artists, and recommendations. Handlebars was used for server-side rendering of views.

**Recommendation Engine:** Implemented a content-based recommendation system that reads the metadata (genre, artist, tags) of the currently playing track and returns similar songs from the catalog in real-time — no external ML library needed, pure logic.

<ul role="list" className="mt-6 space-y-8">
  <li>
    <span>
      <strong className="font-semibold text-blue-100">Streaming & Download:</strong> Songs and albums are served via streaming endpoints so the browser plays audio progressively without waiting for a full download. A separate endpoint handles file downloads.
    </span>
  </li>
  <li>
    <span>
      <strong className="font-semibold text-blue-100">Search:</strong> A search API accepts keyword queries and returns ranked results across songs, albums, and artists using string matching and simple relevance scoring.
    </span>
  </li>
  <li>
    <span>
      <strong className="font-semibold text-blue-100">Digital Wellbeing:</strong> A client-side timer tracks cumulative listening time per session and prompts the user to take a break after a configurable threshold — inspired by YouTube's digital wellbeing feature.
    </span>
  </li>
  <li>
    <span>
      <strong className="font-semibold text-blue-100">Shuffle Mode:</strong> A true shuffle that samples randomly from the full catalog rather than the current playlist — so users always discover something new, independent of listening history.

  </li>
</ul>
