  ## Challenge

  Build a system that can:

  1. Model a network of countries, capitals, and routes as a weighted graph.
  2. Load real data from CSV files — no hardcoded values.
  3. Find the shortest travel path using an efficient algorithm with proper C++ OOP design.
  4. Display the result visually on a map — not just print it to the console.
  5. Use operator overloading and custom comparators correctly within the graph logic.

## Solution

<AppSolution href={appData.url}>

**OOP Architecture:** Designed a clean class hierarchy — `Country`, `Capital`, `Route`, and `Graph` — using encapsulation and proper separation of concerns. Each class owns its data and exposes only the necessary interface.

<ul role="list" className="mt-6 space-y-8">
  <li>
    <span>
      <strong className="font-semibold text-blue-100">CSV File Parsing:</strong> A custom file reader parses country, capital, and route data from CSV files at runtime and populates the graph dynamically — no hardcoded geography in the source code.
    </span>
  </li>
  <li>
    <span>
      <strong className="font-semibold text-blue-100">Dijkstra's Algorithm:</strong> Implemented using a priority queue with a custom comparator (operator overloading) on the Route class to correctly order nodes by cumulative travel distance — achieving O((V + E) log V) time complexity.
    </span>
  </li>
  <li>
    <span>
      <strong className="font-semibold text-blue-100">Path Reconstruction:</strong> After Dijkstra's runs, a backtracking step reconstructs the full optimal path from source to destination by tracing predecessor nodes stored during traversal.
    </span>
  </li>
  <li>
    <span>
      <strong className="font-semibold text-blue-100">Map Visualization:</strong> The C++ program outputs the shortest path as a JSON coordinate list, which a lightweight HTML + JavaScript page reads to render the route on a Leaflet.js map — giving a real visual result, not just a list of city names.
    </span>
  </li>
</ul>

</AppSolution>
