    ### DAX Formula Patterns

    The three most important DAX patterns for any Data Analyst.

    #### Pattern 1 — Running Total

    ```dax
    Running Total Revenue =
    CALCULATE(
        [Total Revenue],
        FILTER(
            ALL(date[date]),
            date[date] <= MAX(date[date])
        )
    )
    ```

    #### Pattern 2 — Top N with RANKX

    ```dax
    City Revenue Rank =
    RANKX(
        ALL(markets[markets_name]),
        [Total Revenue],
        ,
        DESC,
        DENSE
    )
    ```

    Use with a visual-level filter: keep only where `City Revenue Rank <= 5` to show top 5 cities dynamically.

    #### Pattern 3 — Dynamic Target vs Actual

    ```dax
    Target Achievement % =
    DIVIDE(
        [Total Revenue],
        [Revenue Target],
        0
    )
    ```

    ```dax
    Status =
    IF(
        [Target Achievement %] >= 1,
        "✅ On Track",
        "❌ Below Target"
    )
    ```

    Use the Status measure as a color rule in conditional formatting on table visuals — no manual color-coding needed.

    ### Data Modelling — Star Schema

    The Sales Insights project uses a star schema — one central fact table surrounded by dimension tables. This is the standard for fast, flexible analytics.

    #### Schema Design:

    ```
    FACT TABLE
    ─────────────────────────────────────────
    transactions
      ├── product_code (FK → products)
      ├── customer_code (FK → customers)
      ├── market_code (FK → markets)
      ├── order_date (FK → date)
      ├── sales_amount
      ├── sales_qty
      └── profit_margin

    DIMENSION TABLES
    ─────────────────────────────────────────
    products       → product_code, product_type
    customers      → customer_code, custmer_name, customer_type
    markets        → market_code, markets_name, zone
    date           → date, year, month_name, cy_date
    ```

    #### Power Query Cleaning Steps:

    1. Filter `sales_amount <= 0` — removes invalid or test transactions
    2. Normalize currency: convert USD amounts to INR
       ```
       = Table.AddColumn(Source, "norm_sales_amount",
           each if [currency] = "USD" then [sales_amount] * 75
           else [sales_amount])
       ```
    3. Extract Year and Month from date column for time intelligence
    4. Remove duplicates from dimension tables on the key column

    **Relationship type:** All dimensions connect to the fact table via single-direction, many-to-one relationships — this prevents ambiguous filter propagation and keeps DAX measures predictable.
